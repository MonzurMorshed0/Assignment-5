What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

What is the event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique in the DOM where you attach a single event listener to a parent element instead of multiple listeners to child elements. This works by taking advantage of event bubbling, where events propagate up from the target element to its ancestors.
Benefits: Performance: Reduces the number of event listeners. Dynamic Elements: Handles events for elements added after the listener is set. Cleaner Code: Centralizes event handling.

How do you manipulate an element's attributes and styles using the DOM?

Manipulating Attributes
Get an Attribute: Use getAttribute to retrieve a value. Set an Attribute: Use setAttribute to change or add a value. Remove an Attribute: Use removeAttribute to delete an attribute.
Manipulating Styles
Set a Style: Use the style property to change CSS styles. Get a Style: Access the style property to retrieve the current style. Remove a Style: Use removeProperty to delete a specific CSS style.
